# Dynamic Visual SLAM using a General 3D Prior

## 0. 一句话结论
利用通用 3D 几何先验区分静态/动态区域，实现鲁棒的动态场景 SLAM。

## 1. 核心问题
在动态场景中，传统 SLAM 的静态世界假设被打破，运动物体导致位姿估计偏差和地图污染。已有动态 SLAM 方法要么依赖语义分割（需要额外模型），要么基于光流/多视图一致性（计算开销大、在纹理缺失区域失败）。核心问题是：如何用一种通用的 3D 几何先验——无需语义标签或专用分割网络——来识别动态像素并排除其对 SLAM 的影响？

## 2. 核心方法
该方法利用 3D 高斯 splatting 的几何一致性作为"通用 3D 先验"：
1. **3D Prior 构建**：用 3DGS 从多帧构建场景的 3D 表示，利用 GS 的空间分布和密度作为几何先验——静态区域在时间上累积一致的 GS 密度，动态区域则出现矛盾。
2. **动态检测**：通过比较新帧与 3D 先验的重建残差（reprojection error / photometric inconsistency），检测不符合 3D 先验的区域。动态物体不会在 3D 先验中累积一致的几何，因此残差显著高于静态区域。
3. **SLAM Pipeline**：仅使用与 3D 先验一致的区域（静态区域）进行位姿优化和地图更新，动态区域被 mask 排除。
4. **Iterative Refinement**：3D 先验本身会随着 SLAM 的进行不断更新，形成闭环：更准确的位姿 → 更好的 3D 先验 → 更好的动态检测。

这种方法不依赖任何语义先验，纯粹从几何一致性出发区分动静。

## 3. 实验结果
- 在 Dynamic TUM 和 ScanNet 上动态场景位姿估计精度显著优于静态假设方法
- 无需额外的语义分割或光流模块，推理开销与标准 [[gs-slam/monocular/GS-SLAM]] 相当
- 在物体运动剧烈的场景下，动态 mask 准确率超过 90%
- 局限：对缓慢移动的物体（与相机运动混合时）检测率下降

## 4. 与 SkelGS-SLAM 的关联
- **可借鉴点**：3D 先验驱动的动态检测机制可以直接集成到 SkelGS-SLAM 中——anchor 结构天然支持"哪些 GS 在时间上一致"的判断，可作为更精细的动态掩码生成器
- **差异**：该方法用通用 3D 先验，SkelGS-SLAM 的 anchor 结构本身就隐含了几何一致性先验，可能不需要额外的动态检测模块
- **融合方向**：将 anchor 的时空一致性作为动态先验信号——如果一个 anchor 的关联 GS 在不同帧间无法维持一致的 3D 位置，则判定为动态区域。这比通用 3D 先验更高效、更精细
