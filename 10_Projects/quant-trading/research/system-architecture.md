# 系统规格说明书

## 系统目标

建立可复现的量化研究 + 模拟交易系统，通过 2-3 个月纸面交易验证流程稳定性。

## 架构

```
用户入口 (微信/Obsidian/CLI)
    ↓
Hermes (常驻中枢)
    ↓
任务路由器
    ↓
├── Hermes 处理：信息收集、文档归档、日报、简单决策
└── Codex 处理：代码开发、审计、复杂分析
    ↓
量化系统 (~/code/quant-research/)
    ↓
人工确认层（最终决策）
```

## 职责边界

### Hermes
- 信息入口（微信/Obsidian/CLI）
- 新闻抓取与摘要
- 文档归档到 Obsidian
- 策略想法整理成实验卡片
- 模拟交易日报生成
- 任务路由（判断是否调用 Codex）

### Codex
- 代码开发（数据管道、因子、回测、模拟交易）
- 代码审计（未来函数、数据泄漏、幸存者偏差）
- 实验复现（固定种子、可复现配置）
- 框架重构（notebook → 可测试模块）

### 禁止
- LLM 直接下实盘单
- LLM 接触 API key
- 新闻情绪直接转交易指令
- 未审计策略上线

- 只做 A 股现货
